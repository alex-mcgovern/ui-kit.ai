# Button

A button allows a user to perform an action, with mouse, touch, and keyboard
interactions.

## Usage

### `variant`

Boondoggle has three variants of buttons, `primary`, `secondary`, and `tertiary`. If
no `variant` is specified, a `primary` button will be rendered.

<Canvas of={examples.variant} />

### `isDestructive`

When `isDestructive` is set to `true` the Button will styled in red, to denote a destructive action.

<Canvas of={examples.isDestructive} />

### `isDisabled`

A button can be disabled by setting the `isDisabled` prop to `true`. This will
prevent the button from being interacted with. Opacity is reduced to provide
some visual feedback that the button is disabled.

<Unstyled>
    <div className="grid grid-cols-1 md:grid-cols-2 gap-2">
        <Canvas className="!my-2" of={examples.isDisabled} />
        <Canvas className="!my-2" of={examples.isDestructiveIsDisabled} />
    </div>
</Unstyled>

### `isIcon`

When `isIcon` is set to `true` the Button will be styled to be square with a fixed
height & width. This should be used in conjunction with passing an icon
component to the `children` prop.

If you aren't passing text to the `children` prop, ensure you are passing an `aria-label` attribute, so the Button is correctly labelled to assistive technologies

<Canvas of={examples.isIcon} />

### `slotLeft` / `slotRight`

A decorative node (e.g. an icon) to render on the left or right side of the
Button. When a node is passed, the padding on the corresponding side is
slightly reduced to maintain visual balance.

<Unstyled>
    <div className="grid grid-cols-1 md:grid-cols-2 gap-2">
        <Canvas className="!my-2" of={examples.slotLeft} />
        <Canvas className="!my-2" of={examples.slotRight} />
    </div>
</Unstyled>
